**Мобильное приложение для инспекторов Пожнадзора г. Стерлитамака**  
Автоматизация учёта объектов надзора, контроль средств пожарной безопасности и своевременное оповещение о нарушениях.

> ⚠️ Данный репозиторий содержит только **frontend-часть** (React Native). Для работы требуется бэкенд-сервер (Node.js/Python) и Firebase Cloud Messaging.

---

## 📱 О проекте

Приложение разработано в рамках всероссийского хакатона «Кибер 102» для **Управления надзорной деятельности и профилактической работы (Пожнадзор) г. Стерлитамака**.

**Цели:**
- Повышение эффективности работы инспекторского состава
- Своевременное предупреждение нарушений в области пожарной безопасности
- Создание единой цифровой базы подконтрольных объектов
- Ускорение процесса проверок и формирования отчётности

---

## 🚀 Основной функционал

### Для всех авторизованных пользователей
- Поиск и фильтрация объектов по наименованию, адресу, типу, классу пожарной опасности
- Детальная карточка объекта (все данные, документы, схемы, ответственные лица)

### Для администратора
- **Управление объектами**: добавление, редактирование, архивация
- **Управление ответственными лицами**: назначение, смена, история назначений
- **Документооборот**: загрузка PDF/JPG/PNG (схемы эвакуации, планы размещения инвентаря, декларации) с версионностью и контролем сроков действия
- **Панель управления (Дашборд)**: статистика, список последних изменений
- **Отчёты**: выгрузка реестра объектов в Excel/PDF с фильтрацией

### Средства пожарной безопасности
- Учёт огнетушителей (инвентарный номер, тип, место установки, даты заправки и следующей проверки)
- Учёт пожарных кранов, гидрантов, щитов с фиксацией дат проверок

### Уведомления и контроль сроков
- Push-уведомления (FCM) за 30/14/3 дня до истечения срока
- Ежедневные уведомления о просрочках
- Дублирование на email ответственного инспектора

### Отчётность и проверки
- Формирование акта проверки с замечаниями и фотофиксацией
- Генерация стандартных отчётов

### Технические возможности
- **Офлайн-режим**: просмотр данных и внесение результатов проверок без сети с последующей синхронизацией
- **Интеграция с картами** (Яндекс.Карты / 2ГИС): отображение объектов и построение маршрута
- **Многоуровневая аутентификация**: логин/пароль + PIN-код
- **Разграничение прав доступа** (редактирование — только закреплённому инспектору)
- **Шифрование данных** на устройстве и при передаче

## 🛠 Технический стек (Frontend)

| Компонент | Технология |
|-----------|-------------|
| **Фреймворк** | React Native 0.72 |
| **Язык** | TypeScript |
| **Навигация** | React Navigation 6 |
| **State Management** | Zustand / React Context API |
| **Локальная БД** | Realm / SQLite (офлайн-синхронизация) |
| **HTTP-клиент** | Axios |
| **Карты** | react-native-yandexmaps / react-native-2gis |
| **Камера/фото** | react-native-vision-camera + react-native-image-picker |
| **Push-уведомления** | Firebase Cloud Messaging (FCM) |
| **Хранение данных** | react-native-encrypted-storage (шифрование) |
| **PIN-код/биометрия** | react-native-keychain + react-native-simple-pin |
| **Стилизация** | Tailwind CSS (NativeWind) + StyleSheet |
| **Формы/валидация** | React Hook Form + Zod |
| **Генерация отчётов** | react-native-pdf + react-native-xlsx |
